Gentoo (Linux)

Use flags

While you don't *have* to use these use-flags, you most definitly want to (on a user system):

cryptsetup - Setting and using encrypted drives

policykit - (Today's polkit), for authorization (unpriv. to priv.)

udisks - Normal users can mount drives (requires policykit)

X, xcb, opengl - Graphics with Xorg

ssl - Encryption stuff

threads, openmp - Parallalism

unicode, icu, cjk - International symbols/software, ...

alsa, pulseaudio - Audio (while you don't need to use pulseaudio, it's a pain without pulseaudio)

a52, aac, wavpack, encode, vorbis, flac, lame - Audio and video codecs

jpeg png imagemagick gif - The same for images

Init system

I'm using systemd because most Linux software package target it. The standard init system is OpenRC (it's fun to work with and provides more choices and is closer to the UNIX software idiom). You can also choose basic init sytems like *runit* (Void Linux, anyone?).

Custom Linux Kernel

That's the real fun using Gentoo: You can easily setup your custom Linux Kernel and even live directly on the (*stable*) git releases. The real fun starts if you want to omit things like *initramfs* (initial ramdisk to provide firmware), increase the security of the kernel or optimize the boot time.
